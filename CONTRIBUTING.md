# Contribuer

1. Créer une branche depuis `main`.
2. Garder chaque module focalisé sur une responsabilité.
3. Ne pas appeler directement `window.grist` hors de `src/grist/`.
4. Exécuter `npm run check` avant chaque push.
5. Documenter toute nouvelle colonne dans `docs/DATA_MODEL.md`.
6. Ajouter une ligne à `CHANGELOG.md` pour une fonctionnalité visible.
7. Ne jamais ajouter de données CIEDS réelles ou de secrets au dépôt.
